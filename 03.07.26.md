# Amazon CloudWatch

Amazon CloudWatch is an AWS monitoring service used to track the performance and health of AWS resources. It collects metrics, logs, and generates alerts based on defined conditions.

---

# CloudWatch Features

* Real-time monitoring
* Collects performance metrics
* Stores application logs
* Creates alarms
* Dashboard visualization
* Integrates with AWS services

---

# CloudWatch Components

## Metrics

Metrics are numerical values used to monitor the performance of AWS resources.

**Examples**

* CPU Utilization
* Network In / Out
* Disk Read & Write
* Memory Usage (with CloudWatch Agent)

---

## Logs

CloudWatch Logs store log files from AWS services and applications.

**Examples**

* EC2 logs
* Application logs
* Lambda logs
* Web server logs

---

## Alarms

CloudWatch Alarms monitor metrics and trigger actions when a specified threshold is reached.

**Example**

If **CPU Utilization > 80%**, CloudWatch can:

* Send notifications
* Trigger Auto Scaling
* Invoke a Lambda function

---

# Creating a CloudWatch Alarm

**Steps**

1. Open **CloudWatch** from the AWS Console.
2. Go to **Alarms → Create Alarm**.
3. Select a metric (e.g., CPU Utilization).
4. Set the threshold value.
5. Configure notifications using **Amazon SNS**.
6. Review and create the alarm.

# Interview Questions

### 1. What is Amazon CloudWatch?

Amazon CloudWatch is an AWS monitoring service that collects metrics, logs, and events to monitor the performance and health of AWS resources.

### 2. What are CloudWatch Metrics?

Metrics are numerical measurements used to monitor resource performance, such as CPU utilization and network traffic.

### 3. What are CloudWatch Logs?

CloudWatch Logs store log files generated by AWS services and applications for monitoring and troubleshooting.

### 4. What is a CloudWatch Alarm?

A CloudWatch Alarm monitors a metric and performs an action when a defined threshold is reached.

### 5. What is Amazon SNS?

Amazon SNS (Simple Notification Service) is used to send notifications, such as emails or messages, when a CloudWatch alarm is triggered.

### 6. Give some advantages of CloudWatch.

* Real-time monitoring
* Automatic alerts
* Centralized log management
* Better resource monitoring
* Improved application reliability
