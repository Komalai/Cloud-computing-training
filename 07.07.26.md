# Amazon EBS

Amazon Elastic Block Store (EBS) is a block storage service used with EC2 instances. It provides persistent storage, so data remains available even after an instance is stopped.

### Features

* Persistent storage
* High performance
* Scalable capacity
* Encryption support
* Snapshot backups
* Attach and detach volumes

---

# Common Uses of EBS

* Operating system storage
* Databases
* File systems
* Boot volumes
* Application storage
* Backups

---

# Types of EBS Volumes

| Volume Type   | Use Case                    |
| ------------- | --------------------------- |
| **gp3 / gp2** | General-purpose workloads   |
| **io1 / io2** | High-performance databases  |
| **st1**       | Big data and log processing |
| **sc1**       | Infrequently accessed data  |

---

# Creating an EBS Volume

**Steps**

1. Open **AWS Console**.
2. Go to **EBS → Volumes**.
3. Click **Create Volume**.
4. Select the volume type, size, and Availability Zone.
5. Click **Create Volume**.

**Example**

```text id="d12-1"
Volume Type : gp3
Size : 10 GB
Availability Zone : ap-south-1a
```

---

# Attaching an EBS Volume

Steps performed:

1. Select the created volume.
2. Choose **Actions → Attach Volume**.
3. Select the EC2 instance.
4. Enter the device name.
5. Click **Attach**.

# EBS Snapshots

A snapshot is a backup of an EBS volume that can be used to restore data or create new volumes.

**Steps**

1. Select the EBS volume.
2. Click **Actions → Create Snapshot**.
3. Enter a description.
4. Create the snapshot.

# Amazon EBS vs Amazon S3

| Amazon EBS                               | Amazon S3                          |
| ---------------------------------------- | ---------------------------------- |
| Block Storage                            | Object Storage                     |
| Attached to EC2                          | Accessible over the internet       |
| Low latency                              | Highly scalable                    |
| Used for operating systems and databases | Used for files, backups, and media |

# Interview Questions

### 1. What is Amazon EBS?

Amazon EBS is a block storage service that provides persistent storage for Amazon EC2 instances.

### 2. What is an EBS Snapshot?

An EBS Snapshot is a backup of an EBS volume that can be used for recovery or creating new volumes.

### 3. What is the difference between EBS and S3?

* **EBS** provides block storage and is attached to EC2 instances.
* **S3** provides object storage for files and data over the internet.

### 4. Can an EBS volume be detached from an EC2 instance?

Yes, an EBS volume can be detached from one EC2 instance and attached to another instance in the same Availability Zone.

### 5. Why are snapshots important?

Snapshots help in backup, disaster recovery, and restoring data when required.

---

# Key Points

* EBS provides persistent block storage for EC2.
* Snapshots are used for backup and recovery.
* Volumes can be attached or detached from EC2 instances.
* Different EBS volume types are available for different workloads.
* EBS is best suited for operating systems, databases, and application storage.
