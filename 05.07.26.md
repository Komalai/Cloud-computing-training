# Connecting to an EC2 Instance

An EC2 instance can be accessed remotely using **SSH (Secure Shell)**. It provides a secure connection between the local system and the EC2 instance.

### Requirements

* Running EC2 instance
* Public IP Address
* Key Pair (`.pem`)
* Port **22 (SSH)** enabled in the Security Group

---

# SSH Connection

**Ubuntu**

```bash id="d11-1"
ssh -i <key-name>.pem ubuntu@<public-ip>
```

Example:

```bash id="d11-2"
ssh -i mykey.pem ubuntu@13.234.56.78
```

**Amazon Linux**

```bash id="d11-3"
ssh -i mykey.pem ec2-user@13.234.56.78
```

# Web Servers

A web server is software used to host and deliver web pages over HTTP or HTTPS.

**Popular Web Servers**

* Apache
* Nginx

---

# Apache Web Server

Apache is a widely used open-source web server that is easy to install and configure.

### Installation

```bash id="d11-7"
sudo apt install apache2 -y
```

Start Apache:

```bash id="d11-8"
sudo systemctl start apache2
```

Enable at boot:

```bash id="d11-9"
sudo systemctl enable apache2
```

Check status:

```bash id="d11-10"
sudo systemctl status apache2
```

---

# Nginx Web Server

Nginx is a lightweight, high-performance web server commonly used for web hosting and reverse proxy.

### Installation

```bash id="d11-11"
sudo apt install nginx -y
```

Start Nginx:

```bash id="d11-12"
sudo systemctl start nginx
```

Enable at boot:

```bash id="d11-13"
sudo systemctl enable nginx
```

Check status:

```bash id="d11-14"
sudo systemctl status nginx
```

---

# Accessing the Web Server

Steps performed:

1. Allow **HTTP (Port 80)** in the Security Group.
2. Copy the EC2 Public IP.
3. Open a browser.
4. Visit:

```text id="d11-15"
http://<EC2-Public-IP>
```

The default Apache or Nginx page should appear if the installation is successful.

---

# Apache vs Nginx

| Apache                     | Nginx                     |
| -------------------------- | ------------------------- |
| Easy to configure          | High performance          |
| Process-based              | Event-driven              |
| Better for dynamic content | Better for static content |
| Higher memory usage        | Lower memory usage        |

---

# Key Points

* SSH is used to securely connect to EC2 instances.
* Apache and Nginx are popular web servers.
* Package installation is done using `apt`.
* Port **22** is required for SSH and **80** for web access.
* Web servers can be managed using `systemctl`.
