## Cloud Computing

Cloud computing is a method of using computing resources such as servers, storage, databases, and software over the internet. It eliminates the need to maintain expensive physical infrastructure and allows users to access resources whenever required.

### Key Benefits

* Flexible resource allocation
* Cost-efficient pricing
* Easy access from any location
* Reliable backup and recovery
* Scalable infrastructure

---

## Cloud Services Models

### Infrastructure as a Service (IaaS)

Offers virtual computing resources like servers, networking, and storage that users can configure according to their needs.

**Examples**

* Amazon EC2
* Microsoft Azure Virtual Machines
* Google Compute Engine

### Platform as a Service (PaaS)

Provides developers with an environment to build, test, and deploy applications without worrying about server management.

**Examples**

* AWS Elastic Beanstalk
* Google App Engine
* Heroku

### Software as a Service (SaaS)

Allows users to access software directly through a web browser without installing it on their devices.

**Examples**

* Gmail
* Google Docs
* Microsoft 365
* Spotify

## Amazon Web Services (AWS)

AWS is a cloud platform developed by Amazon that provides numerous services for computing, networking, storage, databases, and security. It is commonly used because it supports applications of all sizes while offering high reliability and flexible pricing.

### AWS Services Discussed

| AWS Service | Use                          |
| ----------- | ---------------------------- |
| EC2         | Launch virtual machines      |
| S3          | Store files and objects      |
| IAM         | Manage users and permissions |
| RDS         | Managed relational databases |
| VPC         | Private networking           |
| Lambda      | Execute code without servers |
| CloudWatch  | Monitor AWS resources        |

## Identity and Access Management (IAM)

IAM is an AWS security service that helps control access to cloud resources. It allows administrators to create users, organize them into groups, assign permissions, and define access policies based on different roles.

### Main Features

* User creation and management
* Group-based permissions
* Role assignment
* Access policies
* Integration with MFA

---

## Multi-Factor Authentication (MFA)

MFA improves account protection by requiring users to complete an additional verification step after entering their password. This can be done using an authenticator application, OTP, or a hardware security key.

### Why MFA is Important

* Prevents unauthorized logins
* Adds an extra layer of security
* Protects AWS resources from compromised passwords
