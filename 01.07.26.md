# Docker Image

A Docker Image is a template that contains everything required to run an application. It is used to create Docker containers.

**Contents**

* Application code
* Runtime
* Libraries
* Dependencies
* Configuration files

---

# Dockerfile

A Dockerfile is a text file that contains instructions for building a Docker image.

### Common Instructions

| Instruction | Purpose                |
| ----------- | ---------------------- |
| `FROM`      | Base image             |
| `WORKDIR`   | Set working directory  |
| `COPY`      | Copy project files     |
| `RUN`       | Execute commands       |
| `EXPOSE`    | Open a container port  |
| `CMD`       | Default command to run |

---

# Creating a Dockerfile

Steps performed:

1. Open **Visual Studio Code**.
2. Create a project folder.

```text id="w0mk8x"
docker-demo
```

3. Create a file named:

```text id="u7tb2e"
Dockerfile
```

Example:

```dockerfile id="8dmp1n"
FROM nginx:latest

COPY . /usr/share/nginx/html

EXPOSE 80
```

---

# Building a Docker Image

Navigate to the project folder and run:

```bash id="zqf4sm"
docker build -t my-nginx-image .
```

**Explanation**

* `docker build` → Build the image
* `-t` → Assign image name
* `.` → Current directory

Check available images:

```bash id="r4kg7b"
docker images
```

---

# Running the Image

Create and run a container:

```bash id="x0qb3m"
docker run -d -p 8080:80 --name mycontainer my-nginx-image
```

Open the application:

```text id="y0r1jz"
http://localhost:8080
```

For AWS EC2:

```text id="n2ev6a"
http://<EC2-Public-IP>:8080
```

# Docker Hub Repository

Steps:

1. Sign in to Docker Hub.
2. Click **Create Repository**.
3. Enter the repository name.
4. Choose **Public** or **Private**.
5. Create the repository.

Example:

```text id="2jp1cs"
username/my-nginx-image
```

---

# Docker Login

Login from the terminal:

```bash id="b71tm0"
docker login
```

Enter your Docker Hub username and password.

---

# Tagging an Image

Before pushing an image, assign a Docker Hub tag.

```bash id="m5z4cn"
docker tag my-nginx-image username/my-nginx-image:latest
```

---

# Pushing an Image

Upload the image to Docker Hub.

```bash id="8hc1ra"
docker push username/my-nginx-image:latest
```

Verify the uploaded image by opening your Docker Hub repository.

# Important Commands

| Command                                      | Purpose             |
| -------------------------------------------- | ------------------- |
| `docker build -t myimage .`                  | Build an image      |
| `docker images`                              | List images         |
| `docker run -d -p 8080:80 myimage`           | Run a container     |
| `docker login`                               | Login to Docker Hub |
| `docker tag myimage username/myimage:latest` | Tag an image        |
| `docker push username/myimage:latest`        | Push image          |
| `docker pull username/myimage:latest`        | Pull image          |

---

# Key Points

* Dockerfile automates image creation.
* Images can be built locally using `docker build`.
* Docker Hub is used to store and share images.
* Tagging is required before pushing an image.
* Docker images can be downloaded and reused on any system.
