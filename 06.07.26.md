# Connecting to an EC2 Instance

SSH (Secure Shell) is used to securely access an EC2 instance from a local machine.

### Requirements

* Running EC2 Instance
* Public IPv4 Address
* Key Pair (`.pem`)
* Port **22** enabled in the Security Group

---

# SSH Commands

**Ubuntu Instance**

```bash
ssh -i <key-name>.pem ubuntu@<public-ip>
```

Example:

```bash
ssh -i mykey.pem ubuntu@13.234.56.78
```

**Amazon Linux Instance**

```bash
ssh -i mykey.pem ec2-user@13.234.56.78
```

# Web Servers

A web server hosts websites and delivers web pages to users through HTTP or HTTPS.

**Common Web Servers**

* Apache
* Nginx

---

# Apache Web Server

Apache is an open-source web server commonly used for hosting websites.

### Installation & Management

Install:

```bash
sudo apt install apache2 -y
```

Start:

```bash
sudo systemctl start apache2
```

Enable at startup:

```bash
sudo systemctl enable apache2
```

Check status:

```bash
sudo systemctl status apache2
```

---

# Nginx Web Server

Nginx is a lightweight web server known for its speed and efficient performance.

### Installation & Management

Install:

```bash
sudo apt install nginx -y
```

Start:

```bash
sudo systemctl start nginx
```

Enable at startup:

```bash
sudo systemctl enable nginx
```

Check status:

```bash
sudo systemctl status nginx
```

---

# Accessing the Web Server

Steps:

1. Allow **HTTP (Port 80)** in the EC2 Security Group.
2. Copy the EC2 Public IP.
3. Open a browser and enter:

```text
http://<EC2-Public-IP>
```

The default Apache or Nginx page will be displayed if the server is running.

---

# Apache vs Nginx

| Apache                     | Nginx                       |
| -------------------------- | --------------------------- |
| Open-source web server     | High-performance web server |
| Process-based architecture | Event-driven architecture   |
| Better for dynamic content | Better for static content   |
| Higher memory usage        | Lower memory usage          |

---

# Key Points

* SSH provides secure remote access to EC2 instances.
* Apache and Nginx are widely used web servers.
* `apt` is used to install packages on Ubuntu.
* `systemctl` is used to manage services.
* Port **22** is required for SSH and **Port 80** for web access.
