# Amazon VPC

Amazon VPC (Virtual Private Cloud) is a networking service that allows users to create a private network in AWS. It helps organize cloud resources securely and gives control over IP addresses, subnets, and routing.

### Why VPC?

* Provides network isolation
* Improves security
* Controls internet access
* Allows custom network configuration

---

# VPC Components

### Subnet

A subnet is a smaller network inside a VPC.

**Public Subnet**

* Connected to the Internet Gateway
* Used for web servers
* Accessible from the internet

**Private Subnet**

* No direct internet access
* Used for databases and backend services
* More secure than public subnets

---

### Internet Gateway (IGW)

An Internet Gateway connects a VPC to the internet and allows communication between AWS resources and external users.

---

### Route Table

A Route Table defines how network traffic is routed within the VPC.

| Destination | Target           |
| ----------- | ---------------- |
| Local       | Local VPC        |
| 0.0.0.0/0   | Internet Gateway |

---

### Network ACL (NACL)

A Network ACL is a subnet-level firewall that controls incoming and outgoing traffic.

**Features**

* Works at subnet level
* Supports Allow and Deny rules
* Stateless

---

### NAT Gateway

A NAT Gateway allows instances in a private subnet to access the internet without making them publicly accessible.

**Uses**

* Software updates
* Package installation
* Secure outbound internet access

---

# EC2 and RDS Architecture

For better security, the database should remain inside a private subnet instead of being exposed to the internet.

```text
Internet
    │
Internet Gateway
    │
Public Subnet
    │
EC2 Instance
    │
Private Subnet
    │
RDS Database
```

**Advantages**

* Better security
* Database remains private
* Reduced attack surface
* Secure communication between EC2 and RDS

# Creating a VPC

Steps performed:

1. Open **AWS Management Console**.
2. Search for **VPC**.
3. Click **Create VPC**.
4. Enter VPC name and CIDR block.
5. Create Public and Private Subnets.
6. Attach an Internet Gateway.
7. Create Route Tables.
8. Configure Network ACLs and Security Groups.

**Example**

```text
VPC Name: MyVPC
CIDR Block: 10.0.0.0/16

Public Subnet : 10.0.1.0/24
Private Subnet: 10.0.2.0/24
```

---

# Launching an EC2 Instance

Steps:

1. Open the EC2 Dashboard.
2. Click **Launch Instance**.
3. Select an AMI and Instance Type.
4. Choose the created VPC.
5. Select the required subnet.
6. Configure the Key Pair and Security Group.
7. Launch the instance.

---

# Best Practices

* Deploy web servers in Public Subnets.
* Keep databases in Private Subnets.
* Use Security Groups for instance security.
* Use NACLs for subnet protection.
* Use a NAT Gateway for secure internet access from private instances.
