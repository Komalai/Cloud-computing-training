# Amazon S3

Amazon S3 (Simple Storage Service) is AWS's object storage service used to store and retrieve files over the internet. It is commonly used for backups, website hosting, and application storage.

### Features

* Scalable object storage
* High durability
* Secure data storage
* Versioning support
* Static website hosting
* Lifecycle management

### Common Applications

* File storage
* Backups
* Static websites
* Media storage
* Data archiving

---

# S3 Basics

### Bucket

A bucket is a container that stores objects (files) in Amazon S3.

**Bucket Naming Rules**

* Must be globally unique
* Use lowercase letters, numbers, and hyphens
* No spaces or uppercase letters

**Example**

```text
my-training-bucket
```

### Object

An object is any file stored inside a bucket.

**Examples**

* Images
* Videos
* Documents
* HTML files
* PDF files

---

# Creating an S3 Bucket

Steps performed:

1. Open **AWS Management Console**.
2. Search for **S3**.
3. Click **Create Bucket**.
4. Enter the bucket name and region.
5. Configure versioning, encryption, and public access settings.
6. Click **Create Bucket**.

**Example**

```text
Bucket Name: my-training-bucket
Region: ap-south-1 (Mumbai)
```

---

# Uploading Files

Steps:

1. Open the bucket.
2. Click **Upload**.
3. Select files.
4. Review settings.
5. Click **Upload**.

---

# Hosting a Static Website

Amazon S3 can host static websites containing HTML, CSS, JavaScript, and images.

Steps:

1. Upload website files.
2. Enable **Static Website Hosting**.
3. Set `index.html` as the index document.
4. Configure public access and bucket policy.
5. Open the generated website endpoint.

# S3 Storage Classes

| Storage Class              | Purpose                                   |
| -------------------------- | ----------------------------------------- |
| S3 Standard                | Frequently accessed data                  |
| Intelligent-Tiering        | Automatic cost optimization               |
| Standard-IA                | Infrequently accessed data                |
| One Zone-IA                | Low-cost storage in one Availability Zone |
| Glacier Instant Retrieval  | Archived data with fast access            |
| Glacier Flexible Retrieval | Long-term backups                         |
| Glacier Deep Archive       | Lowest-cost archival storage              |

---

# Lifecycle Configuration

Lifecycle rules automatically move or delete objects after a specified period.

**Example Rule**

* 30 Days → Standard-IA
* 90 Days → Glacier
* 365 Days → Delete Object

**Benefits**

* Reduces storage cost
* Automates data management
* Minimizes manual work

---

# AWS Snow Family

AWS Snow Family devices are used to transfer large amounts of data between on-premises systems and AWS when network transfer is slow or impractical.

### Snowcone

* Small and portable
* Suitable for remote locations
* Supports edge computing

### Snowball Edge

* Large storage capacity
* Secure data transfer
* Local processing support

### Snowmobile

* Truck-sized data transfer service
* Designed for exabyte-scale migration
* Used by large enterprises

---

# Advantages of AWS Snow Family

* Faster large-scale data transfer
* Secure migration
* Less dependency on internet bandwidth
* Supports edge computing
